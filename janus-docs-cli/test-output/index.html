User Management API

Comprehensive user management operations with authentication and profile management

Protocol

Unix Domain Sockets with JSON messaging

Version

1.0.0

Channels

2 available

Commands

7 total

Protocol Overview

This API uses Unix Domain Sockets for inter-process communication with JSON-based messaging and 4-byte length prefixes.

Message Format

{
  "id": "uuid-v4-string",
  "channelId": "channel-identifier", 
  "command": "command-name",
  "args": {
    "key": "value"
  },
  "timeout": 30.0,
  "timestamp": "2025-07-29T10:50:00.000Z"
}

Response Format

{
  "commandId": "uuid-from-request",
  "channelId": "channel-identifier",
  "success": true,
  "result": {
    "data": "response-data"
  },
  "timestamp": "2025-07-29T10:50:01.000Z"
}

Authentication

Authentication is handled through Unix socket permissions and optional channel-specific authentication mechanisms.

Error Handling

All errors follow a standardized format with specific error codes and detailed messages.

{
  "commandId": "uuid-from-request",
  "channelId": "channel-identifier", 
  "success": false,
  "error": {
    "code": "ERROR_CODE",
    "message": "Human readable message",
    "details": "Additional context"
  },
  "timestamp": "2025-07-29T10:50:01.000Z"
}

User Service

Core user management operations including registration, authentication, and profile management

Create User

Create a new user account with validation and security checks

Channel: user-service Command: create-user Timeout: 30s

Arguments

Username
string required

Unique username for the account

Pattern: ^[a-zA-Z0-9_]{3,50}$

Min length: 3

Max length: 50

Email Address
string required

User's email address for notifications and recovery

Pattern: ^[\w\.-]+@[\w\.-]+\.[a-zA-Z]{2,}$

Password
string required

Secure password meeting complexity requirements

Min length: 8

Max length: 128

User Role
string optional

User's system role determining permissions

Default: "user"

Allowed values: "admin", "moderator", "user", "guest"

User Profile
object optional

Extended user profile information

Response

Type: object

User creation result with generated ID and status

Model: CreateUserResponse

Error Codes

VALIDATION_FAILED USERNAME_EXISTS EMAIL_EXISTS PASSWORD_TOO_WEAK SECURITY_VIOLATION
